OpenCore7.0升级

折腾黑苹果有一段时间了，一直想尝试自己升级一下opencore，但是因为懒到现在才尝试。

下面进入正题，opencore其核心设置都是储存在 config.plist，这个文件中的其本质上其实是一个 XML 文件，任意一个专业的代码编辑器都可以操作它。它拥有 XML 的所有基本特征，内容上主要就是使用标签的树状结构。

config.plist 在 opencore 中主要作用是储存各项设置的内容，当 opencore升级时，其开发团队会通过修改 config.plist 的文件结构，增减功能区或对应的选项，以支持新版本扩展出来的功能。

升级的方有两种，一种是通过opencore configurtion，将原来的配置项一项一项的移到sample.plist

另一种是利用代码比较工具找到sample.plist和你的config.plist不同项直接修改

这里当然是采用第二种，谁要一项项移动啊喂！

下面，开工！！！

由于不太熟悉结构，我先对比了两个版本的sample.plist

可以看到只有8个差异部分，改动部分很少

然后直接在config.plist里更改

最后，把关键文件替换一下

  • /EFI/BOOT/BOOTx64.efi：OpenCore 的引导文件，必须替换
  • /EFI/OC/Driver/OpenRuntime：OpenCore 的功能库，必须替换
  • /EFI/OC/Bootstrap/Bootstrap.efi：如果使用了 Bootstrap，则需要替换
    • 注：OpenCore 自 0.6.6 版本开始删除了 Bootstrap，如果之前你使用了 Bootstrap，需要先在 config 中关闭，重启计算机并重置 NVRAM，最后再升级 OC 文件
  • /EFI/OC/Driver/AudioDxe.efi：如果使用了开机声音，则需要替换
  • /EFI/OC/Driver/OpenCanopy：如果使用了官方主题服务，则需要替换
    • 需要注意OpenCore7.0以后的主题文件格式变了，需要在/EFI/OC/Resources/Image里新建文件夹放入主题图片以区别不同的主题，并在PickerVariant里填入对应文件夹地址
  • /EFI/OC/OpenCore.efi

最后的最后，重启！如果进不去的话，就凉凉了（这提醒了我们备份的重要性，在进行上述操作之前一定要备份好EFI文件）。
