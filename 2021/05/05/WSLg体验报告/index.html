WSLg体验报告

WSLg是什么？

相信不少人都听说过wsl(Windows Subsystem for Linux)即适用于Linux的Windows子系统，这是微软开发的一个为在Windows 10和Windows Server 2019上能够原生运行Linux二进制可执行文件（ELF格式）的兼容层。简而言之就是能在Windows系统上运行linux终端而不需要虚拟机。

而WSLg就是在wsl的基础上加了GUI，其实这个大饼微软在一年前就画了出来，但最近才在GitHub开源了预览版本。根据微软的介绍

WSLg致力于使Linux GUI应用程序在Windows上自然而自然地使用。从集成到启动的“开始”菜单中开始到出现在任务栏中，alt-tab体验到在Windows和Linux应用程序之间启用剪切/粘贴，WSLg都可以利用Windows和Linux应用程序提供无缝的桌面体验和工作流程。

类似于macos下的parallel desktop的融合模式，wslg能够直接在开始菜单和任务栏里启动linux下的gui程序。

官方预览图

安装

微软不愧是最佳linux开发者，怀着激动的心情我火速看了看安装指南。

安装要求系统要在Windows 10 Insider Preview build 21362+，我一直都是dev版本（日常使用确实不推荐，bug太多），这个跳过。

同时微软推荐安装GPU驱动以在wslg中使用vgpu以支持机器学习，看了一下支持列表有我的Radeon 500 Series。

由于我之前已经开启了wsl2，现在只用升级wsl --update并重启wsl --shutdown即可

测试

我首先安装了一个Ubuntu子系统，测试了一下推荐的软件

gedit

Nautilus

Edge

VScode

基本上都能正常使用，剪切板同步正常，音频能够输出。

问题

缩放支持差

wslg默认不开启dpi缩放，默认的高dpi对小屏幕不友好，而根据相关Issue（#23）开启缩放后，画面糊的根本不能用！

开启缩放后的vscode

默认不支持中文

按官方教程来无法显示中文，还得自己安装中文包

总结–未来可期

WSLg凭借着和Windows的深入集成（开始菜单等）让Windows和Linux之间不再有一条鸿沟，虽然现在还有很多bug，但是等它开发到稳定版以后，凭借着和Linux生态的兼容，Windows说不定就要成为最佳Linux桌面发行版了🤪。
