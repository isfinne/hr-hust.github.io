背靠阿里云——我是这样优化博客的

得益于一个在阿里云备了案的域名，我可以使用阿里云的一系列产品来优化我的博客，价格也算不上很贵。下面来罗列一下我使用的服务。

全站加速 DCDN

全站加速（Dynamic Route for Content Delivery Network）是阿里云自主研发的融合了动态加速和静态加速技术的CDN产品。该产品一站式解决了页面动静态资源混杂、跨运营商、网络不稳定、单线源站、突发流量、网络拥塞等诸多因素导致的响应慢、丢包、服务不稳定的问题，提升全站性能和用户体验

全站加速与CDN对比

对比项CDN全站加速
支持资源类型仅支持静态内容加速。同时支持静态内容和动态内容加速。
加速方式将您服务器上的静态内容缓存在CDN节点上供您就近访问。静态内容使用CDN加速。动态内容通过智能路由、协议优化等动态加速技术快速访问您的服务器源站获取。
源站适配建议您对服务器源站的动静态内容进行分离，静态内容使用CDN加速，动态内容不使用CDN加速。您无需对服务器源站上的资源进行改造，全站加速会智能区分动静态内容并分别加速。

可以看到全站加速支持静态内容和动态内容加速。

开启后速度确实有了很大的提升，基本上所以的值都降了下了。

教程可见使用阿里云CDN全站加速gitpages

需要注意的是部署的时候有个坑，如果你的域名之前已经绑定github pages，请删除你的CNAME记录，再将CANNE记录指向阿里云的CDN，然后阿里云的CDN源站再选择username.github.io。

其实就相当于访问的时候：你的域名–>阿里云CDN–>github pages

静态资源优化

对于我这种不会CSS、HTML、JavaScript的小白，优化静态资源就只能靠压缩了(笑

简单介绍一下我用的工具：

  • hexo-all-minifier 一个all-in-one的hexo插件

    安装方法也很简单，在博客根目录执行一下代码就安装好啦

    npm install hexo-all-minifier --save

    在博客根目录的站点配置文件_config.yml里添加配置项

    html，css，js压缩我都选择了开启

  • 图片压缩我使用的是 TinyPNG，相比 ImageOptim 等工具 TinyPNG 的压缩率更高、有损压缩时图片细节丢失最少。反正大家都说好用

阿里云OSS将图像处理为webp

完成上面的动作后，发现还是有点小问题

WebP 是 Google 最新开发的新图像格式，旨在以可接受的视觉质量为无损和有损压缩提供较小的文件大小。有损模式下比 JPEG 小 25% - 34%，无损模式下较 PNG 小 26%。

看到大佬的WebP 图片优化在 Hexo 上的最佳尝试 ，我也很是心动，无奈大佬的方法只能把图片放在本地，不支持图床，就暂时搁浅了。

后来想起来阿里云的OSS不是有个图像处理功能吗，就去瞧了瞧，结果发现居然可以转换成webp，简直太激动了，马上开始了尝试，操作也很简单，创建一个格式后，在url后面加上相应的后缀就可以引用处理后的图像了。

比如我是https://static.finne.top/img/post2/3.png!webp

一番操作下来，图片加载速度肉眼可见的提升
